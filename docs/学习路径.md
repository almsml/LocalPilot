# LocalPilot 学习路径

> 按这个顺序阅读代码和文档，从零理解整个项目。预计 **3-5 天** 完整走完。

---

## Day 1: 理解架构（先看全景，不碰代码）

### 1.1 读设计文档（30 分钟）

先读 `LocalPilot-规划.md` 的前三章，建立全局认知：

| 章节 | 重点 | 时间 |
|------|------|------|
| §一 — 项目定位 | 一句话叙事 + 核心差异化——**这个项目解决什么问题** | 5 min |
| §二 — 系统架构 | 架构图 + 数据流（设备上线→心跳→任务提交→故障迁移） | 10 min |
| §三 — 技术选型 | Agent/Controller/Dashboard 各自用什么语言，为什么 | 10 min |
| §四 — 实施计划 | 5 个 Phase 各做什么，产出什么 | 5 min |

### 1.2 读 README（10 分钟）

`README.md` — 快速开始命令 + 目录结构 + 技术栈表格。

### 1.3 跑起来（30 分钟）

```bash
# 三终端启动，看一眼 Dashboard 就行
# 终端 1
cd controller && go run ./cmd/controller/
# 终端 2
cd agent && LOCALPILOT_HOSTNAME=my-pc go run ./cmd/agent/
# 终端 3
cd dashboard && npm run dev
# 打开 http://localhost:5173
```

**验收自己：能用自己的话讲清楚"Controller 是什么、Agent 是什么、Dashboard 是什么，它们之间怎么通信"。**

## Day 2: 理解通信层（proto + gRPC）

### 2.1 Proto 定义（30 分钟）

系统契约——所有组件通信的"宪法"。

| 文件 | 读什么 | 
|------|--------|
| `proto/localpilot/v1/agent.proto` | `DeviceService`（Register/Heartbeat/Deregister）和 `TaskExecutionService`（Execute/CancelTask）。**重点看 service 定义和 message 字段注释** |
| `proto/localpilot/v1/task.proto` | `Task` 模型、`TaskStatus` 状态机（QUEUED→ASSIGNED→RUNNING→COMPLETED）、`TaskSharding` 分片策略 |
| `proto/localpilot/v1/discovery.proto` | mDNS 配置常量（当前为空） |

### 2.2 gRPC 实现（60 分钟）

**Agent 侧（gRPC client + server）：**

| 文件 | 关注点 |
|------|--------|
| `agent/internal/transport/client.go` | `DeviceServiceClient` 如何连接 Controller、调用 Register/Heartbeat/Deregister |
| `agent/internal/transport/server.go` | `TaskExecutionServer` 如何接收 Controller 的 Execute 请求、流式推送 LogChunk |

**Controller 侧（gRPC server + client）：**

| 文件 | 关注点 |
|------|--------|
| `controller/internal/transport/grpc.go` | `DeviceServiceServer` 如何实现 Register/Heartbeat/Deregister、extractIP 从 peer 提取真实 IP |
| `controller/internal/transport/client.go` | `TaskExecutionClient` 如何连接 Agent、调用 Execute（Phase 3 调度后使用） |

**验收自己：能画出"Agent 调用 Controller.Register(DeviceInfo) → Controller 返回 device_id → Agent 启动心跳 → 每 5 秒 Heartbeat"的时序图。**

## Day 3: 理解 Agent（设备端）

按启动顺序阅读 `agent/cmd/agent/main.go`，然后逐个深入模块。

### 3.1 启动流程（20 分钟）

`agent/cmd/agent/main.go` — 按 1-9 步阅读，每步对应一个 `internal/` 模块。

### 3.2 核心模块（90 分钟）

| 模块 | 文件 | 重点 |
|------|------|------|
| **配置** | `internal/config/config.go` | 环境变量如何覆盖默认值、`LOCALPILOT_HOSTNAME` 的作用 |
| **监控** | `internal/monitor/monitor.go` | 跨平台系统信息采集（gopsutil + 平台 fallback） |
| **心跳** | `internal/heartbeat/heartbeat.go` | Ticker 循环 → sendHeartbeat → 超时/错误处理 → Phase 4 重连信号 |
| **mDNS** | `internal/discovery/mdns.go` | hashicorp/mdns 广播 `_localpilot._tcp`、TXT 记录携带 agent_port |
| **沙箱** | `internal/sandbox/sandbox.go` | `Sandbox` 接口 → `ProcessSandbox`（os/exec）→ DockerSandbox（TODO） |
| **执行器** | `internal/executor/executor.go` | `Execute()` 启动 goroutine → bufio.Scanner 逐行读取 → LogChunk channel → gRPC stream |

**验收自己：能画出 Agent 的 goroutine 结构图——哪些在后台独立运行（心跳、gRPC serve、mDNS、executor task goroutine），主 goroutine 在做什么。**

## Day 4: 理解 Controller（控制中心）

### 4.1 启动流程（15 分钟）

`controller/cmd/controller/main.go` — 按 1-7 步阅读。

### 4.2 核心模块（120 分钟）

| 模块 | 文件 | 重点 |
|------|------|------|
| **设备注册表** | `internal/registry/device.go` | 内存 + SQLite 双层存储、RWMutex 并发模型、UUID 设备 ID、重注册检测、LoadDevices 重启恢复 |
| **心跳检测** | `internal/registry/health.go` | ticker → checkHealth → ONLINE→UNHEALTHY(15s)→OFFLINE(30s) → Phase 4 离线回调触发任务迁移 |
| **任务存储** | `internal/job/store.go` | SQLite jobs 表、args/env JSON 序列化、CRUD |
| **任务队列** | `internal/job/queue.go` | channel FIFO 队列、Dequeue 阻塞等待 |
| **任务管理器** | `internal/job/manager.go` | **这是最核心的文件**——SubmitJob → 持久化 → 入队 → worker 取任务 → 调度器选设备 → gRPC Execute → 收 LogChunk → 更新状态。还有 Phase 4 的 MigrateTasksFromDevice |
| **调度器** | `internal/scheduler/scheduler.go` + `scorer.go` | SelectDevice → Score(5 维度加权) → 排序 → 返回最高分设备 |
| **HTTP API** | `internal/api/router.go` + `device_handler.go` + `job_handler.go` | gin 路由注册、CORS 中间件、设备/任务端点 |
| **mDNS** | `internal/discovery/mdns.go` | hashicorp/mdns Lookup 监听 `_localpilot._tcp` |

**验收自己：能画出"用户提交 echo hello → Controller 做了什么 → Agent 做了什么 → 日志怎么回到 Dashboard"的完整数据流。**

### 5.2 面试话术（30 分钟）

读 `LocalPilot-规划.md` §六（面试话术库），重点准备 5 个问题：

1. **"为什么 Agent 和 Controller 都用 Go？"** — 统一语言栈降低认知负担，proto 代码共享，交叉编译简单，goroutine 天然适合心跳/gRPC
2. **"调度器怎么选设备的？"** — 五维加权打分模型（CPU×0.35 + 内存×0.25 + 架构×0.20 + 延迟×0.10 + GPU×0.10）
3. **"设备离线了怎么办？"** — 30 秒确认 OFFLINE → 回调触发任务迁移 → 任务重新入队 → 调度到其他设备
4. **"最大的技术挑战是什么？"** — 系统中立——网络层/调度层/执行层/UI 层四层协同
5. **"和 K8s 比好在哪？"** — K8s 面向同构节点，LocalPilot 面向极度异构的个人设备，更轻量

## 附录: 每个模块的"一句话总结"

| 模块 | 一句话 |
|------|--------|
| `config` | 从环境变量加载配置，Hostname 可被 LOCALPILOT_HOSTNAME 覆盖 |
| `monitor` | gopsutil 跨平台采集 CPU/内存/温度，平台降级兜底 |
| `heartbeat` | Ticker 每 5 秒发 Heartbeat，连续失败 6 次触发重连信号 |
| `discovery` (Agent) | hashicorp/mdns 广播 _localpilot._tcp，TXT 记录含 agent_port |
| `sandbox` | Sandbox 接口 → ProcessSandbox(os/exec) → DockerSandbox(TODO) |
| `executor` | goroutine 执行命令，bufio.Scanner 逐行推 LogChunk 到 channel |
| `transport` (Agent) | gRPC client(DeviceService) + server(TaskExecutionService) |
| `registry` | 内存 map + SQLite 双层，RWMutex 保护，UUID 设备 ID |
| `health` | ticker 检查心跳，15s→UNHEALTHY，30s→OFFLINE→触发迁移回调 |
| `job/store` | SQLite jobs 表 CRUD，args/env 以 JSON 存储 |
| `job/queue` | channel FIFO，Dequeue 阻塞等待 |
| `job/manager` | SubmitJob→持久化→入队→worker→调度→gRPC Execute→收日志 |
| `scheduler` | 五维加权打分，选最高分 ONLINE 设备 |
| `api` | gin 路由，CORS，设备/任务 REST 端点 |
| `discovery` (Controller) | hashicorp/mdns Lookup 发现 Agent，记录日志 |
